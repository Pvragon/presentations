From a conversation, to a package

A conversation with Bradd this week prompted a review of the best practices we've currently implemented for agent design — pulling from current literature and our own work into one coherent picture.

  • DOE architecture — directive · orchestration · execution
  • Anthropic's two long-running-agent harness papers — the planner / generator / evaluator pattern
  • Karpathy's recent CLAUDE.md observations — behavioral guidance for coding agents
  • Ralph-loop pattern + progressive disclosure — the runtime shape and the context-loading shape
  • Our own layered-memory design — rooted in OS memory hierarchy thinking; vendor-independent agent identity in the spirit of Anthropic's Memory Tool

Pulled together into a single portable handoff package.

What's in the package

  • Why each pattern exists — the failure mode it defends against
  • How they compose — DOE → Ralph-loop harness → multi-agent council → layered memory + progressive disclosure → behavioral rules
  • A primitive translation table — works on Claude Code, Codex, Cursor, Aider, MCP-only, or any in-house harness
  • 12 copy-pasteable starter templates — AGENTS.md, MEMORY.md, SKILL.md, feature_list.json, council rubric, Ralph-loop pseudocode, and more

Point your agent at this package and tell it to emulate the setup.
