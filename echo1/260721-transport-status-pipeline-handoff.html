02 The one limitation you must carry

This reads like a flow, but it is a count inventory. A true Sankey requires that every ride sit in exactly one lane — and that breaks in exactly one place.

Why mutual exclusivity breaks

7,078 rides are billed in error. They sit in the Billed terminal and belong to an upstream gate they never actually cleared — 2,345 missing a trip log, 2,293 missing a signature, 39 never reviewed, 2,401 billed after the cancel cutoff. In a real Sankey, a ride that fails "Trip documented?" flows out to the DRIVERS lane and cannot reappear downstream. Here it does both. That double membership is the leak.

Two checksums — one holds, one doesn't

Partition / conservation checksum ✓ HOLDS top split  11,405 + 211 + 85,979 = 97,595
funnel     406 + 123 + 3,001 + 1,358 + 81,091 = 85,979
terminals  Σ 7 buckets = 97,595
Flow / gate-pass checksum ✗ BREAKS rides that truly passed each gate ≠ the Billed terminal, off by exactly 7,078 — the billed-in-error set

Every ride lands in exactly one terminal, so the endpoints must re-foot to the cohort — that's your build-integrity gate. The flow checksum fails by precisely the billed-in-error count, which gives you a bonus invariant:

Σ(clean gate-pass flow) − Σ(partition Billed) = −7,078  // if a rebuild's error overlay ≠ this delta, something is mis-bucketed

03 Going forward — when the Sankey becomes real

The billed-in-error set is the entire source of the break — it isn't intrinsic to the lifecycle. The gates are meant to be strictly sequential: reaching "Billed" is only legitimate if the ride already cleared Trip-documented → Reviewed → valid cancel state. When the system enforces that (you physically cannot bill without complete docs, a review, and a valid cancel state), then every ride occupies one lane, the two checksums converge, and a genuine Sankey is valid.

So "no logic issues" is equivalent to "the state machine is enforced." The billed-in-error count is precisely a metric of state-machine violations — zero by construction in a clean-forward world. That's what the Echo1 freeze-gradient (TR → Transport → Trip Log → Billing Review → Claim → Paid) is designed to guarantee.

Keep this honest

Even with perfect enforcement, model the legitimate transitions as their own states or they'll read as violations: recoupments / reversals (billed then clawed back — needs its own terminal) and late-arriving evidence (a held ride that correctly re-enters the flow). Mutual exclusivity holds as long as every legitimate transition is a modeled state.
